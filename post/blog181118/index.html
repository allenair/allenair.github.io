尝试Go语言

原因

最近一段时间在学习使用Go语言，使用Go语言主要基于以下几个原因：

  1. 自己一直使用Java作为主要工作语言，由于是WEB开发也免不了一些Html和Javascript的脚本。与前台展现比较个人更倾向于后台重逻辑和算法方面的能力。
  2. Java本身是一个非常不错的后台语言，但是由于历史包袱它的轻量化比较艰难。另一个更重要的问题是，JVM的体量越来越大，这个虽然在传统服务端不是什么大问题，但是在轻量的服务端（如物联网）则是一个致命的问题。因此Java的模块化，大幅缩减JVM体量，使Java能部分回归最初的嵌入式设计是亟待解决的问题。
  3. 语法轻量化方面，可以尝试Python，它在库的丰富性、语法友好性、快速开发方面有较大优势，特别是在数据挖掘与机器学习领域，快速原型的能力是无可替代的，这点比Java强。但它同时有虚拟机的问题。
  4. Go语言在这方面比较有优势，语法方面比较简单，能快速掌握（有些语法较奇特需要适应一段时间），语法便捷性方面借鉴了Python，是编译型语言最终编译结果是一个平台相关的可执行文件（自带执行环境），部署相当容易，特别适合微服务与轻量服务端场景。
  5. 想象一下部署一个Java Web服务，需要一个JVM环境还需要一个Tomcat应用服务器，Python也相似需要有Python环境。但是Go语言部署的全部就是一个编译后的可执行文件。

体会

谈一点自己的体会。

  1. 其实符合性能需求的还有Rust语言，但是看了一下语法和目前接受程度，还是选择学习Go，今后可以再看一下以便更好的理解系统编程。
  2. 其实编程语言千差万别，真没有必要都掌握，只是需要在需要解决问题的不同层面上掌握一种方案即可，例如，前端HTML是必须的；控制脚本方面可以使用JS，但是基于JS的种种问题（毕竟一周设计出的语言还是太草率了）今后需要看一下TypeScript；企业后端服务Java，.net就不必了，毕竟解决是一样的事情；手机端的原生方案也不必了，无论安卓还是IOS都不是我的菜，贪多嚼不烂，即使有需要我能遇到的场景使用web方式都能解决；数据与算法设计方面python是唯一的；高效服务方面C、Go、Rust均可，鉴于难度还是选择Go。
  3. Go语法有一些别扭的地方需要适应，其他像声明方式、返回err等感觉还好，就是感觉Duck Typing的方式，在看代码的时候有些不便。在Java里面可以通过接口声明与实现能快速理解代码功能，在Go中如果不对接口比较熟悉，看的时候容易混乱。
  4. Go中channel的概念简单，但是应用起来有一定难度，需要通过看一些第三方项目进行体会。
  5. 下一步学习Go中标准库，特别是其中的web、网络编程、io、同步一些库的用法。
  6. 交叉编译非常方便，尝试了在Macos 64位（amd64）机器上编译在树莓派linux 32位（ARM）运行的程序，能方便实现。

GOOS=linux GOARCH=amd64 go build hello.go
上面这段代码直接在命令控制台里面运行就可以生成64位linux的可执行程序。
GOOS：目标操作系统
GOARCH：目标操作系统的架构

OS ARCH S version
linux 386 / amd64 / arm >= Linux 2.6
darwin 386 / amd64 OS X (Snow Leopard + Lion)
windows 386 / amd64 >= Windows 2000
freebsd 386 / amd64 >= FreeBSD 7
