#《区块链革命——分布式自律型社会出现》 ##整体感觉 + 区块链的书分成三类

概要性的，内容比较宽泛，主要讲趋势实际案例少，这是区块链早期书的特点

偏重应用的，目前较多的是金融应用方面的介绍，这也是区块链比较重要的应用领域

偏重技术的，会从区块链的底层技术进行说明，重点从密码学、分布性、信任机制、开发包的应用这几方面进行详细讲述

  • 这本书是偏重应用的，作者是金融领域转件因此金融方面案例较多
  • 区块链最重要解决的是信任的问题，传递“A给B一块钱”的信息是简单的，问题是怎么让大家信任，如果大家都信任了，就可以认为实现了“价值”的实际转移
  • 全部基于区块链的应用，就是基于它的数据不可篡改这一特性展开的，因为有技术保障的信息不可篡改性就保证了数据的真实，因此任何人之间的互动就不需要建立在预先信任的基础之上，这就是Trustless的真谛

##笔记 ####区块链概念 + 区块链解决了传统互联网不能实现的事情。第一，传递货币等有经济价值之物。第二，建立信任机制。 + 区块链解决不信任的方法。第一，不仰仗管理者。第二，保护数据不被篡改。 + PoW（工作量证明）机制，这是共识机制的一种。这个是挖矿的主要工作，就是不停的枚举以便找出特别的“数字”，找到后就可以新建一个区块，同时也获得奖励。 + 拜占庭将军问题是一个协议问题，拜占庭帝国军队的将军们必须全体一致的决定是否攻击某一支敌军。问题是这些将军在地理上是分隔开来的，并且将军中存在叛徒。叛徒可以任意行动以达到以下目标：欺骗某些将军采取进攻行动；促成一个不是所有将军都同意的决定，如当将军们不希望进攻时促成进攻行动；或者迷惑某些将军，使他们无法做出决定。如果叛徒达到了这些目的之一，则任何攻击行动的结果都是注定要失败的，只有完全达成一致的努力才能获得胜利。 + 区块链在互联网领域能做的另一件事，是实现智能合约。“智能合约”是以计算机能理解的形式订立的协议。无须人类判断，通过预先设定好的程序自动执行合约（这种合约也被称为“铁面无私合约”）。智能合约的自动执行，将大大降低现有商业组织或者商业型个人的交易成本，甚至可实现零成本交易。

####区块链应用 + 银行业。区块链率先敲开银行业的大门。致力于开展将区块链技术应用于转账服务的实验。积极探索将区块链应用于银行核心业务——结算系统。 + 证券业。国际知名证券交易机构都完成区块链交易系统测试。证券交易的清算、结算领域，有望借助区块链技术实现工作效率的显著提升。 + 保险业和金融衍生品领域：区块链技术的应用范围正不断扩展到保险业及金融衍生品等领域。 + 重塑金融业格局，科技金融业（Fintech2.0）。 + 其他行业，例如物品追溯。 + 构筑新的社会模式，完全自由的社会，没有中心管理者，类似亚当斯密的完全自由市场经济

##扩展 ####参考《2018区块链白皮书》 + 区块链定义：是一种由多方共同维护，使用密码学保证 传输和访问安全，能够实现数据一致存储、难以篡改、防止抵赖的记 账技术，也称为分布式账本技术。 + 工作方式：各参与方按照事先约定的规则共同存储信息并达成共识。为了防止共识信息被篡改，系统以区块(Block)为单位存储数据，区块之间按照时间顺序、结合密码学算法构成链式 (Chain)数据结构，通过共识机制选出记录节点，由该节点决定最新区块的数据，其他节点共同参与最新区块数据的验证、存储和维护， 数据一经确认，就难以删除和更改，只能进行授权查询操作。 + 使用场景： + 技术架构： + 共识机制：共识层负责协调保证全网各节点数据记录一致性。从业务应用的需求看，共识算法的实现应综合考虑应用环境、性能等诸多要求。共识机制在区块链中扮演着核心的地位，决定了谁有记账的权利，以及记账权利的选择过程和理由，因此一直是区块链技术研究的重点。为提升效率，需在安全性、可靠性、开放性等方面进行取舍。 + 智能合约：一个最重要的漏洞源，图灵完备脚本语言交强大，但存在陷入死循环的可能。为了适应智能合约，需要创造新的合约语言或为现有语言增加形式更为严格的规范和校验，建立完整安全的防护体系。 + 应用场景： + 部署：与云结合的BaaS方式是快速落地实施的一种比较好的方案。
